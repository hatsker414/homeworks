## Задача 1

Архитектор ПО решил проконсультироваться у вас, какой тип БД 
лучше выбрать для хранения определенных данных.

Он вам предоставил следующие типы сущностей, которые нужно будет хранить в БД:

- Электронные чеки в json виде
- Склады и автомобильные дороги для логистической компании
- Генеалогические деревья
- Кэш идентификаторов клиентов с ограниченным временем жизни для движка аутенфикации
- Отношения клиент-покупка для интернет-магазина

Выберите подходящие типы СУБД для каждой сущности и объясните свой выбор.

## Решение задачи 1

1 Электронные чеки в json виде
  В данном случаем подойдет noSQL база где необходимо хранить ключ-значение
2 Склады и автомобильные дороги для логистической компании
  графовые СУБД
3 Генеалогические деревья
  Иерархическая субд - очень похожа не генеалогическое дерево по своей структуре
4 Кэш идентификаторов клиентов с ограниченным временем жизни для движка аутенфикации
  Redis - можно выставлять TTL
5 Отношения клиент-покупка для интернет-магазина
  Думаю сюда подойдет графовая субд. 


## Задача 2

(каждый пункт - это отдельная реализация вашей системы и для каждого пункта надо привести классификацию):

- Данные записываются на все узлы с задержкой до часа (асинхронная запись)
